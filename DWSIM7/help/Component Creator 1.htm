Component Creator Window

The Component Creator Utility contains several tabs to define all necessary properties and a fixed upper part.



Upper fixed part

  1. Path
    Path and filename of user domponent database where the component will be saved to.
     
  2. Search
    Displays file dialog to search for existing user database to save component into.
     
  3. Create New
    Displays file dialog to define name for creating a new user database
     
  4. Replace components with same ID
    Checking this box will cause replacing all data of a component with identical ID already existing in the database. This option is checked as standard and is usually used if you are updating the database with new parameters.
     
  5. Unit System
    This dropdown field allows to select the unit system to display data. For instance if you have the boiling point in "K" you can select "SI-UNnits" and enter the value in K-units. Then you switch back to enter other parameters in other units like "°C".

  6. Compound Type
    Selects between a Standard-type fluid or Black-Oil type fluid. Black-oIl fluids are ones defined to work specifically with the Black-Oil Property Package.

  7. ID
    Identification numer in database. It is only possible to have an ID once. If an existing ID is used this component will overwrite an existing component with the same ID if {4} is checked.
     
  8. Name
    Component name. This name will be displayed in component selection field of DWSIM.

  9. Save component to database
    Saves component parameters into defined database
     

Component properties

Identification

This tab is used to define necessary information to identify the component. 
  1. CAS-Number
    CAS-Number of component. The blue label of that field is a link to the NIST-Webbook database and opens a page with available data of defined CAS-number.
     
  2. Mole Weight
    Molecular weight of component. This font of this parameter is colored in light grey. This means it can't be edited as it is calculated automatically. See also following point {3}.
     
  3. Joback Calculation
    Checkbox to toggle automatic calculation. Checking this box will cause Component Creator to calculate this parameter automatically from other available information (here: atom definition table). If box is not checked the molecular wheight can be edited by the user and will be fixed even if all parameters of that component are recalculated due to new informations beeing available. If the checkbox is rechecked again, editing will be disabled and the value is recalculated by automatic routines from Compound Creator.
    All changes to a value will cause recalculation of other parameters depending on this parameter.
    This is a general principal of compound creator valid for all parameters!
     
  4. Formula
    Formula of component. This text is calculated automatically depending on definition of atoms.
     
  5. SMILES
    The SMILES code is a standard code for representing the structure of any arbitrary chemical formula.
     
  6. PubChemEditor
    This is a link to the PubChem Editor where you can construct the structure of your component. This Editor will give you the SMILES code of your new molecule. To get this even more simple you will find a so called InChi-Key on NIST-Webbook which is an alternative code to SMILES. This InChi-Key can be copied to PubChem editor to create that component and to get the SMILES code.
     
  7. SMILES Render
    As soon as a SMILES code is available the molecule can be rendered and will be displayed in graphic box{8}.
     
  8. Graphic box
    Graphic box containing rendered molecule from SMILES code. This is great for visualisation of molecular structure.
     
  9. DWSIM-File Status
    This field shows saving status of Componet Creator Study File. This file contains all information for defining this component. This includes the parameters itself but additional informations like experimental data and whether a parameter is calculated automatically or edited manually. Every change of information will set saving status to "modified" 

  10. UserDB Status
    This field shows saving status of componet in database. The database only stores the parameters of that component but no additional informations. Only changes to parameters will set saving status to "modified".
